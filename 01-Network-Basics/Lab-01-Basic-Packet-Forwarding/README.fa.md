# آزمایشگاه ۰۱ - انتقال پایه بسته‌ها در شبکه (Basic Packet Forwarding)

## معرفی

هدف این آزمایش، بررسی نحوه انتقال بسته‌ها بین دو شبکه IPv4 مجزا از طریق یک روتر Cisco ISR 4331 است.

در این سناریو تلاش شد تا با بررسی رفتار پروتکل‌های ARP و ICMP، نحوه پردازش فریم‌های اترنت در لایه دوم (Data Link) و بسته‌های IPv4 در لایه سوم (Network) در ارتباطات داخل یک شبکه و بین دو شبکه مجزا درک شود.

این توپولوژی در نرم‌افزار Cisco Packet Tracer پیاده‌سازی شده و از سوئیچ‌های Cisco Catalyst 3560-24PS استفاده شده است که از طریق لینک‌های فیبر نوری مبتنی بر SFP به روتر متصل شده‌اند.

---

## اهداف

- طراحی و پیاده‌سازی یک شبکه شامل دو زیرشبکه (Subnet) مجزا
- پیکربندی آدرس‌های IPv4 روی اینترفیس‌های روتر
- آدرس‌دهی دستی تمامی سیستم‌های انتهایی
- تنظیم Default Gateway برای تمامی دستگاه‌ها
- برقراری ارتباط بین دو شبکه از طریق روتر
- بررسی عملکرد پروتکل ARP در ارتباطات داخل یک Subnet
- بررسی عملکرد ARP هنگام ارتباط با شبکه‌ای دیگر
- مشاهده فرآیند انتقال بسته‌های ICMP در حالت Simulation نرم‌افزار Packet Tracer
- بررسی نحوه یادگیری آدرس‌های MAC توسط سوئیچ‌ها
- بررسی جدول مسیریابی (Routing Table) روتر

## معماری شبکه

این توپولوژی از دو شبکه محلی (LAN) مستقل تشکیل شده است که توسط یک روتر Cisco ISR 4331 به یکدیگر متصل شده‌اند.

- شبکه سمت چپ: `192.168.10.0/24`
- شبکه سمت راست: `192.168.20.0/24`

هر شبکه از طریق سوئیچ‌های Cisco Catalyst 3560-24PS به روتر متصل شده است. وظیفه سوئیچ‌ها، برقراری ارتباط در لایه دوم (Layer 2) و وظیفه روتر، مسیریابی و انتقال بسته‌ها بین دو شبکه در لایه سوم (Layer 3) است.

## خلاصه پیکربندی

در این آزمایش مراحل زیر انجام شد:

1. طراحی و پیاده‌سازی توپولوژی شامل دو زیرشبکه IPv4 مستقل
2. اتصال سوئیچ Cisco Catalyst 3560-24PS به روتر Cisco ISR 4331 از طریق لینک فیبر نوری مبتنی بر SFP
3. اختصاص آدرس IPv4 به تمامی سیستم‌های انتهایی
4. پیکربندی آدرس IP روی اینترفیس‌های روتر
5. فعال‌سازی اینترفیس‌های روتر با دستور `no shutdown`
6. تنظیم Default Gateway روی تمامی سیستم‌ها
7. بررسی ارتباط بین سیستم‌های داخل یک شبکه
8. بررسی ارتباط بین دو شبکه از طریق روتر
9. مشاهده روند انتقال بسته‌های ARP و ICMP در حالت Simulation
10. بررسی جدول مسیریابی روتر و جدول MAC Address سوئیچ‌ها

## بررسی و اعتبارسنجی

در پایان آزمایش، موارد زیر با موفقیت بررسی شدند:

- برقراری ارتباط بین دستگاه‌های موجود در یک Subnet
- برقراری ارتباط بین دستگاه‌های دو Subnet متفاوت
- مشاهده و تحلیل بسته‌های ARP Request و ARP Reply
- مشاهده و تحلیل بسته‌های ICMP Echo Request و Echo Reply
- بررسی فرآیند یادگیری آدرس‌های MAC توسط سوئیچ‌ها
- بررسی مسیرهای متصل مستقیم (Directly Connected Routes) در جدول مسیریابی روتر

## مفاهیم کلیدی آموخته‌شده

این آزمایش به درک بهتر مفاهیم پایه شبکه کمک کرد، از جمله:

- پروتکل ARP وظیفه تبدیل آدرس IPv4 به آدرس MAC را بر عهده دارد.
- درخواست‌های ARP تنها در محدوده Broadcast Domain محلی ارسال می‌شوند.
- هنگام ارسال بسته به شبکه‌ای دیگر، فرستنده MAC مقصد را پیدا نمی‌کند، بلکه ابتدا MAC مربوط به Default Gateway را به دست می‌آورد.
- سوئیچ‌ها فریم‌های اترنت را بر اساس جدول MAC Address خود ارسال می‌کنند.
- روترها بسته‌ها را بر اساس Routing Table مسیریابی می‌کنند، نه بر اساس MAC Address.
- پروتکل ICMP برای بررسی ارتباط انتها به انتها (End-to-End Connectivity) استفاده می‌شود.
- برقراری ارتباط بین دو شبکه متفاوت نیازمند انجام مسیریابی در لایه سوم است.

---

## چالش‌ها و عیب‌یابی

در هنگام پیاده‌سازی این توپولوژی، ابتدا ارتباط فیزیکی بین روتر و سوئیچ برقرار نمی‌شد.

پس از بررسی مشخص شد که علت این مشکل، استفاده از ماژول‌های SFP ناسازگار روی روتر و سوئیچ بوده است.

همچنین مشخص شد که اینترفیس مورد استفاده روی روتر از هر دو رسانه RJ-45 و SFP پشتیبانی می‌کند و لازم است نوع رسانه به صورت دستی با دستور `media-type sfp` مشخص شود؛ در غیر این صورت لینک فیبر نوری فعال نخواهد شد.

این تجربه اهمیت بررسی لایه فیزیکی و تنظیم صحیح اینترفیس‌ها را پیش از شروع عیب‌یابی در لایه‌های بالاتر نشان داد.

## دستورات مهم

### دستورات بررسی وضعیت

```bash
show ip interface brief
show ip route
show arp
show mac address-table
```

### دستورات پیکربندی

```bash
interface GigabitEthernet0/0/0
 ip address 192.168.10.1 255.255.255.0
 no shutdown

interface GigabitEthernet0/0/1
 ip address 192.168.20.1 255.255.255.0
 no shutdown

media-type sfp
```

## آموخته‌های این آزمایش

این آزمایش ارتباط بین عملکرد لایه دوم و لایه سوم شبکه را به‌خوبی نشان داد.

پیش از ارسال هر بسته ICMP، سیستم فرستنده ابتدا باید آدرس MAC مقصد مناسب را پیدا کند. اگر مقصد در همان Subnet قرار داشته باشد، ARP آدرس MAC همان دستگاه را پیدا می‌کند؛ اما اگر مقصد در شبکه‌ای دیگر باشد، ARP ابتدا آدرس MAC مربوط به Default Gateway را به دست می‌آورد و سپس بسته برای انجام مسیریابی به روتر ارسال می‌شود.

همچنین این آزمایش نشان داد که قبل از بررسی مشکلات مربوط به پروتکل‌ها و تنظیمات شبکه، باید از صحت اتصالات فیزیکی، سازگاری تجهیزات و پیکربندی صحیح اینترفیس‌ها اطمینان حاصل کرد.
